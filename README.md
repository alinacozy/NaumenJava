# Домашнее задание 2

Выполнено домашнее задание 2, включающее 5 заданий:

### Задание №1. Работа с массивом
**Вариант 1**: Найти максимальное значение по модулю в массиве.

Заполняет массив случайными числами и находит элемент с максимальным абсолютным значением.

**Входные данные**: количество элементов в массиве n [0, ∞)
**Выходные данные**: в консоль печатается массив и результат поиска

### Задание №2. Работа со списками
**Вариант 1**: Сортировка слиянием (Merge Sort).

Заполняет список `ArrayList<Double>` случайными числами и сортирует его с помощью алгоритма сортировки слиянием (реализованного вручную).

**Входные данные**: количество элементов в списке n [0, ∞)
**Выходные данные**: в консоль печатаются исходный и отсортированный списки

### Задание №3. Stream API
**Вариант 1**: Отфильтровать сотрудников, оставив только тех, кто старше 30 лет.

1. Реализован класс `Employee` с полями:
    - `fullName` (String) - ФИО сотрудника
    - `age` (Integer) - возраст
    - `department` (String) - отдел
    - `salary` (Double) - заработная плата

2. Создан предзаполненный список из 6 сотрудников

3. С использованием Stream API выполняется фильтрация сотрудников старше 30 лет

**Выходные данные**: в консоль печатается отфильтрованный список сотрудников в читаемом виде

### Задание №4. HTTP клиент и JSON
**Вариант 2**: Вывести только значение идентификационной строки приложения.

Выполняет GET-запрос к `https://httpbin.org/user-agent`, получает JSON-ответ и извлекает значение поля "user-agent" с помощью библиотеки Jackson.

**Выходные данные**: в консоль печатается только значение user-agent (например, "Java-http-client/25.0.2")

### Задание №5. Реализация интерфейса Task
**Вариант 3**: Синхронизация файлов между двумя папками.

Реализует интерфейс `Task` для синхронизации содержимого двух папок:
- `start()` - запускает процесс синхронизации в отдельном потоке
- `stop()` - останавливает процесс синхронизации


**Тестирование**:
Класс `TaskTest` создает тестовую структуру файлов и папок, включая "лишние" файлы в целевой папке для проверки полноценной синхронизации.


## Сборка проекта
```bash
mvn clean package
```

## Запуск проекта
```bash
java -jar target/nau-java-1.0-SNAPSHOT.jar
```

При запуске проекта вам будет предложено выбрать число от 1 до 5, соответствующее номеру задания, а также 0 для выхода.

Каждое задание реализовано в отдельном классе с функцией main, таким образом, можно запускать классы напрямую, а можно через общую точку входа - класс Main.
